# Site v2 Design

Date: 2026-06-29

## Context

`site/` is a Vite + React + TypeScript + react-router app that visualizes the agents repo. Current state:

- **Catalog** (`/`): searchable/filterable cards. 68 nodes (17 agents, 35 skills, 11 shared-refs, 5 MCP servers).
- **Workflows** (`/workflows`): hand-authored flowcharts for 5 pipelines (React Flow + dagre).
- **Relationships** (`/graph`): dagre node-graph derived from agent configs. Only 41 edges, so most skills are orphan dots.

Data is generated by `site/scripts/extract-data.ts` into `site/src/data/graph.json`.

The repo also has promptfoo eval infrastructure: `promptfooconfig.yaml` (20 tests), `promptfooconfig.smoke.yaml` (8 tests), and `evals/metrics/token_usage.jsonl` with per-agent `{model, input_tokens, output_tokens, total_cost_usd, duration_s, timestamp}` records, summarized by `evals/scripts/cost-summary.js`.

## Goals

1. Remove the Relationships graph view (sparse, partly redundant with the detail panel).
2. Add an Overview/stats dashboard as the new home page, using repo data + promptfoo metrics.
3. Enrich node details (skill when-to-use + body excerpt, user-invocable flag, GitHub source links, cross-view linking).
4. Fix scaling: add a centered max-width container, retune card/font sizing, tighten responsive breakpoints.

Non-goals: redesigning the workflows flowchart engine; changing the color/theme system; adding a backend.

## Changes

### A. Remove Relationships view
- Delete `src/views/GraphView.tsx` and `src/views/GraphView.css`.
- Remove the `/graph` route from `App.tsx` and the "Relationships" `NavLink` from `Layout.tsx`.
- Remove the "View in Graph" button + `showGraphLink` path from `NodeDetailPanel.tsx`.
- Keep `graph.json` edges: they still power the detail panel's "Connections" list and the Overview stats.

### B. Routing / nav (Structure)
- `/` -> `OverviewView` (new default).
- `/catalog` -> `CatalogView`.
- `/workflows` -> `WorkflowsView`.
- Nav links: Overview, Catalog, Workflows.

### C. Overview dashboard (new `src/views/OverviewView.tsx` + `.css`)
Reads `graph.json` + new `stats` data. Sections:
- **Hero counts**: agents, skills, shared refs, MCP servers, workflows.
- **Model distribution**: agent count by Opus / Sonnet / Haiku / unspecified (chips or simple bars).
- **Category breakdown**: counts per category for agents and for skills.
- **Eval & cost panel**: total eval test count (full + smoke), and a compact per-agent table/bars of avg input tokens, avg output tokens, avg duration, total cost, derived from `token_usage.jsonl` (ported logic from `cost-summary.js`).
- Cards/sections link into Catalog (optionally pre-filtered) and Workflows.
- Empty-state: if eval metrics are absent/empty, render a graceful "no eval data yet" note.

### D. Richer node details (`NodeDetailPanel`, `CatalogView`, `WorkflowsView`)
- **Skills**: show `whenToUse` (parsed from frontmatter) and a short body excerpt; show a `userInvocable` badge when true.
- **GitHub source link**: every node carries `sourcePath` (repo-relative). Panel renders "View source on GitHub ->" using base `https://github.com/liawedwa/agents/blob/main/<sourcePath>`.
- **Cross-view linking**:
  - Agent detail panel: "Used in workflows" list linking to `/workflows` for workflows whose nodes reference that agent.
  - Workflow step click: link to the referenced agent in `/catalog` (e.g. `/catalog?node=agent:builder`), via a small navigate action.

### E. Scaling
- Add a centered max-width wrapper (~1200px) for Overview and Catalog content; Workflows/graph-style full-bleed canvases keep full width.
- Retune `.card` min-width and font sizes so cards aren't oversized on wide screens; tighten spacing.
- Review and tighten the existing `@media (max-width: 768px)` blocks in `CatalogView.css` and `WorkflowsView.css`; add one for the Overview.

### F. Data pipeline (`scripts/extract-data.ts` + `types.ts`)
- Add `sourcePath?: string` to every node:
  - agents -> `agents/<name>.json`
  - skills -> `skills/<dir>/SKILL.md` (or `skill.md`)
  - shared-refs -> `skills/_shared/<file>.md`
  - mcp-servers -> no source path (synthetic)
- Parse skill frontmatter beyond name/description: capture `whenToUse` (frontmatter `when-to-use` or equivalent) and a plain-text `excerpt` (first ~300 chars of body after frontmatter). Detect `userInvocable` if discoverable from frontmatter/metadata; otherwise omit.
- Emit a `stats` object into `graph.json` (or a sibling `stats.json` imported by Overview):
  - `counts`: { agents, skills, sharedRefs, mcpServers, workflows }
  - `modelDistribution`: { [model]: count } over agents
  - `agentCategories` / `skillCategories`: { [category]: count }
  - `evals`: { fullTestCount, smokeTestCount, byAgent: [{ agent, model, runs, avgIn, avgOut, avgDurationS, totalCostUsd }], grandTotalCostUsd, grandRuns }. Test counts come from counting `description:` entries in the two promptfoo configs; byAgent from `token_usage.jsonl`. If the metrics file is missing/empty, emit `evals` with empty `byAgent` and zeroed totals.

## Architecture / units

- `extract-data.ts`: single build-time script, sole writer of `graph.json`. Pure data; no UI.
- `OverviewView`: pure presentation of `stats` + `graph.json`; no data fetching.
- `CatalogView`: search/filter/group; opens `NodeDetailPanel`.
- `NodeDetailPanel`: presentation of one node + its connections + source link; navigation via react-router.
- `WorkflowsView`: flowchart rendering; step click navigates to catalog.

Each view reads the generated JSON directly (import), matching the existing pattern.

## Error handling
- Missing/empty `token_usage.jsonl`: extract script emits zeroed `evals`; Overview shows empty-state note.
- Missing skill frontmatter: fall back to existing behavior (dir name + first paragraph); `whenToUse`/`excerpt` simply omitted.
- Unknown model on an agent: bucket as "unspecified" in model distribution.

## Testing
- `cd site && npm run build` must pass (tsc + vite build) with no type errors.
- `npx tsx scripts/extract-data.ts` (or the project's existing data-gen command) regenerates `graph.json` without errors and includes the new fields + stats.
- Manual smoke: `npm run dev`, verify Overview renders with real counts, Catalog filters work, detail panel shows source link + skill when-to-use, workflow step links navigate, no console errors, no `/graph` route.

## Implementation / dispatch plan
1. **Foundation (first, solo)**: `extract-data.ts` + `types.ts`; regenerate `graph.json` with `sourcePath`, skill `whenToUse`/`excerpt`/`userInvocable`, and `stats`. All other work depends on these fields.
2. **Then 3 parallel sonnet agents**, disjoint file ownership:
   - **Structure**: `App.tsx`, `Layout.tsx`/`Layout.css`, `index.css`; delete `GraphView.*`; routing + nav + max-width/scaling.
   - **Overview**: new `OverviewView.tsx`/`.css` only.
   - **Catalog & details**: `CatalogView.*`, `NodeDetailPanel.*` (richer details, GitHub link, cross-link), `WorkflowsView.*` step->catalog links + remove `showGraphLink` usage.
3. Build + manual smoke; fix any integration gaps.
